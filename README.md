**Port Authority Passenger Demand Analytics**

**Business Problem**

Transit operations require accurate demand visibility to support capacity planning, scheduling, and long-term resource allocation.

**Objective**

Consolidate fragmented operational data into a structured database and analyze historical passenger trends, seasonal patterns, and projected demand growth.

**Project Scope**

- Consolidated 160+ weekly Excel reports into a normalized SQL Server database

- Designed ETL workflows for automated data validation and transformation

- Performed historical trend and seasonal demand analysis

- Developed multi-year passenger growth projections

- Built executive Power BI dashboard for performance monitoring

**Key Insights**

- Identified consistent seasonal ridership peaks and troughs

- Observed long-term passenger growth trend

- Quantified demand volatility across carriers

- Provided forward-looking projections to support capacity planning discussions

**Business Impact**

- Improved reporting efficiency by eliminating manual reconciliation

- Enhanced demand visibility for operational decision-making

- Supported strategic planning through data-driven forecasting

**Tools Used**
SQL Server, SSIS, Power BI, Excel
